# FAQ

Эти вопросы задают часто.

<details>
  <summary>Переустановил лицензию. Все сделал по инструкции. Все-равно версия free...!</summary>

Возможные причины:

- Неверное название домена. `domain.ru` и `www.domain.ru` — это разные домены.
- Не скачали новый архив с лицензией, а установили из старого.
- Неправильно перезаписали файл лицензии. Например, переименовали каталог с большой буквы.
- Скачивать лицензию надо с того ресурса, где приобретали модуль. Ресурсы обновляются не синхронно.

</details>

<details>
  <summary>Я заинтересован в покупке модуля, не будет ли он конфликтовать с модулем DreamFilter?</summary>

Модуль не конфликтует ни с **DreamFilter**, ни с **megaFilter**, ни с каким другим фильтром. Он не работает с фронтендом, не взаимодействует с фильтрами, а только с базой данных, приводя в порядок **Атрибуты**, которые затем используют фильтры.

</details>

<details>
  <summary>Я сейчас могу указать только тестовый домен, можно будет дополнить позже?</summary>

Можно. В любое время, если количество доменов и их характеристики не противоречат условиям выдачи [лицензии](/license/license-install.md).

</details>

<details>
  <summary>У меня перестал модуль работать. Переезжал на новый хостинг, может из-за этого!?</summary>

Попробуйте:

- Очистить кэш магазина.
- Очистить кэш и куки браузера.
- Обновить модификаторы.
- Выставить заново и сохранить настройки.

</details>

<details>
  <summary>Как сделать слияние значений атрибута?</summary>

Способов, аналогичных способам слияния атрибутов с помощью технологий Drag-and-Drop, контекстного меню или командных клавиш, в модуле не предусмотрено. 
Во-первых, чтобы не усложнять модуль, во-вторых, потому что точно такой же эффект может быть получен при обычном переименовании значений.

По сути, слияние значений или шаблонов — это и есть групповое переименование. Но этот механизм уже реализован. Просто отредактируйте **Значение** или **Шаблон**.

Более подробно о редактировании в разделе [Редактирование](/module-features/editing.md).

</details>

<details>
  <summary>Какой алгоритм действий после парсинга?</summary>

Модуль не предполагает действий по определенному алгоритму, но он предлагает определенный функционал, облегчающий ручную работу, и набор инструментов, позволяющий выполнять некоторые действия в пакетном режиме.

Одним из таких инструментов является инструмент [**Атрибуты категорий**](/tools/category-attributes.md).

:::info
В настройках установим [**`Cпособ`**](/settings/product.md) — **Добавлять дежурный только в пустые**.
Для других вариантов этой настройки результат действия очевиден.
:::

Обозначения:

- `C` — категория
- `P` — товар
- `A` — атрибут
- `V` — значение атрибута
- `D` — дежурный шаблон

Допустим, в категории `C1` есть товар `P1` с **Атрибутом** `A1`, принимающим **Значение** `V1`.
Вы добавили в этот товар **Атрибут** `A2` со **Значением** `V2`, и после парсинга добавились еще два товара `P2` и `P3` со своими **Атрибутами** и соответствующими **Значениями**.

| ![Работа с незаполненными Дежурными шаблонами](/img/tutorial/attribute_category_1.svg) |
|:--:|
| *Диаграмма 1 – Работа с незаполненными Дежурными шаблонами* |

Наша цель — унификация **Атрибутов категории**, т.е. мы хотим, чтобы каждый товар категории `C1` имел одинаковый набор **Атрибутов**.
Применим инструмент **"Атрибуты категорий"**.

:::warning
Некоторые значения оказались пустыми, так как модуль "не знает", что подставлять. Например, **Атрибут** `A1` может принимать **Значения** `V1` или `V4`. Какое выбрать?
:::

Для устранения неоднозначности существуют **Дежурные шаблоны**. Если они заполнены, результат работы инструмента будет таким:

| ![Работа с заполненными Дежурными шаблонами](/img/tutorial/attribute_category_2.svg) |
|:--:|
| *Диаграмма 2 – Работа с заполненными Дежурными шаблонами* |

Теперь все **Значения** заполнены.

</details>

<details>
  <summary>Можно ли допилить модуль, чтобы атрибуты выводились на главной странице под фотографией товара?</summary>

Нет. Этот модуль разработан для админки и с фронтендом не работает. Некоторые возможности могут появиться в будущих продвинутых версиях.
Этот вопрос стоит задавать разработчикам шаблона (дизайна).

</details>
